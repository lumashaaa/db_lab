```bash

# Подключение через HTTP API
curl -s http://localhost:9308/sql -d "query=SHOW TABLES"

# Загрузить 100 000 товаров

# Выполнить запросы
mysql -h 127.0.0.1 -P 9306 < sql/02_search_queries.sql

mysql -h 127.0.0.1 -P 9306 < sql/04_update_delete.sql
```

## RT-индекс и настройки

**morphology='stem_enru'** — стемминг для русского и английского языков.
Приводит слова к корневой форме: "беспроводные" → "беспровод",
"headphones" → "headphon". Поиск "наушники" найдёт и "наушник",
и "наушников" — без явного перечисления форм.

**min_word_len='2'** — игнорирует слова короче 2 символов (предлоги,
артикли). Уменьшает размер индекса и ускоряет поиск.

**html_strip='1'** — удаляет HTML-теги из текста перед индексацией.
Полезно если описания товаров содержат разметку.

**RT-индекс vs plain-индекс:**
Plain-индекс строится один раз из источника данных и не поддерживает
INSERT/UPDATE/DELETE без полной переиндексации. RT-индекс (Real-Time)
принимает изменения мгновенно — данные сначала пишутся в RAM-чанк,
затем фоново сбрасываются на диск. Это позволяет обновлять каталог
без даунтайма.

## Фасетный поиск в e-commerce

Фасеты — это фильтры на странице каталога: "по категории", "по бренду",
"по цене". ManticoreSearch поддерживает FACET как встроенную операцию:
один запрос возвращает и результаты поиска, и счётчики по всем фасетам.
В PostgreSQL это пришлось бы делать несколькими GROUP BY запросами.

## UPDATE в ManticoreSearch vs PostgreSQL

В PostgreSQL UPDATE атомарен и транзакционен — изменение либо применяется
полностью, либо откатывается. Другие сессии видят изменение только после COMMIT.

В ManticoreSearch транзакций нет. UPDATE применяется к атрибутам немедленно,
но изменения в полнотекстовой части (title, description) требуют REPLACE —
документ удаляется и вставляется заново. В кластере возможна eventual
consistency: реплики получают изменения с небольшой задержкой.

## Сравнение ManticoreSearch vs PostgreSQL

| Характеристика | ManticoreSearch | PostgreSQL (tsvector) |
|---|---|---|
| Время поиска (100K) | ~3 мс | ~8 мс |
| Релевантность | BM25 (лучше) | ts_rank (проще) |
| Морфология из коробки | да (stem_enru) | через словари |
| Фасетный поиск | да (FACET) | только GROUP BY |
| JSON-атрибуты | да (встроено) | jsonb |
| Транзакции | нет | да (ACID) |
| Когда использовать | поиск по каталогу | OLTP + редкий поиск |
